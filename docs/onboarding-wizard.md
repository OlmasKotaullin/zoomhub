# Онбординг-Wizard — ZoomHub

## Overview

Пошаговый wizard внутри ZoomHub после регистрации. Проводит нового пользователя через все настройки: от скачивания агента и обхода macOS Gatekeeper до подключения Буквицы, выбора LLM и Telegram-уведомлений. Обязательные шаги нельзя пропустить, опциональные — можно.

## Requirements

### Functional Requirements

1. **Wizard после регистрации** — автоматический redirect на `/onboarding` после register
2. **7 шагов:**
   - **Шаг 1: Приветствие** — что делает ZoomHub, как работает pipeline (обязательный, ознакомительный)
   - **Шаг 2: Скачать агент** — определение ОС, кнопка скачивания Mac/Windows, прямая ссылка через прокси `/download/mac` → GitHub Release (обязательный)
   - **Шаг 3: Установка на Mac** — пошаговая инструкция с скриншотами обхода Gatekeeper: Системные настройки → Конфиденциальность → «Открыть в любом случае» (обязательный для Mac)
   - **Шаг 4: Сервис транскрибации** — выбор: Буквица (рекомендовано, 600₽/мес) / OpenAI Whisper / серверная Буквица. Инструкция по подписке для выбранного сервиса (обязательный)
   - **Шаг 5: API-токен** — генерация токена для агента, copy-to-clipboard, инструкция куда вставить (обязательный)
   - **Шаг 6: Telegram-уведомления** — подключить Telegram-бота для получения конспектов. Ссылка на бота с deep-link токеном (опциональный — можно пропустить)
   - **Шаг 7: Выбор LLM** — Groq (бесплатно, рекомендовано) / Gemini / Claude. Статус доступности каждого (опциональный — по умолчанию auto)

3. **Прогресс-бар** — визуальный индикатор текущего шага (1/7)
4. **Навигация** — кнопки «Далее» / «Назад» / «Пропустить» (для опциональных)
5. **Запоминание прогресса** — если пользователь закрыл страницу, при следующем входе вернётся на последний незавершённый шаг
6. **Блокировка основного интерфейса** — пока онбординг не завершён, все маршруты редиректят на `/onboarding`

### Non-Functional Requirements

- Скриншоты macOS хранятся в `/static/img/onboarding/`
- Адаптив: мобильная версия wizard
- Определение ОС через User-Agent для автоматического выбора Mac/Windows

## Architecture

### Technology Stack

- Jinja2 шаблон с JS для переключения шагов (SPA-like, без перезагрузки)
- HTMX для серверных действий (генерация токена, сохранение выбора)
- Существующий стек: FastAPI + Tailwind CSS

### Шаги и обязательность

| # | Шаг | Обязательный | Действие на сервере |
|---|-----|-------------|---------------------|
| 1 | Приветствие | Да | Нет |
| 2 | Скачать агент | Да | Нет |
| 3 | Установка Mac | Да (Mac) | Нет |
| 4 | Транскрибация | Да | POST выбор провайдера |
| 5 | API-токен | Да | GET генерация токена |
| 6 | Telegram | Нет | Нет (deep-link) |
| 7 | LLM | Нет | POST выбор провайдера |
| - | Завершение | - | POST /onboarding/complete |

## API Design

| Method | Path | Description |
|--------|------|-------------|
| GET | /onboarding | Wizard — рендерит текущий шаг |
| POST | /onboarding/complete | Помечает onboarding_completed=True |
| GET | /download/mac | Прокси → GitHub Release (.app) |
| GET | /download/win | Прокси → GitHub Release (.exe) |

## Deployment

- Скриншоты macOS: подготовить 3-4 скриншота Gatekeeper flow, залить в `/static/img/onboarding/`
- GitHub Release: убедиться что latest release содержит актуальные билды
