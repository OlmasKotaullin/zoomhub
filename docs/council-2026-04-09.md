## Состав консилиума
- **CTO** — архитектура, безопасность, масштабирование

## Контекст
ZoomHub — веб-сервис для автоматической обработки Zoom-записей с AI. Стек: FastAPI + PostgreSQL на Fly.io, LLM fallback chain (Gemini→Groq→Claude), транскрипция через Буквицу. 0 внешних пользователей, только основатель (Алмаз Гатауллин — маркетинг-консультант, экс-CMO Skyeng).

## CTO: Ключевые находки

### P0 — Критические (блокируют запуск)

**1. Безопасность native_api.py — 15 эндпоинтов без user_id фильтрации.**
`list_meetings`, `meeting_detail`, `chat_history`, `list_folders`, `create_folder`, `delete_folder`, `clear_chat`, `delete_meeting`, `update_meeting`, `retry_meeting`, `resummarize_meeting` — ни один не фильтрует по user_id. Любой авторизованный пользователь может видеть/удалять встречи всех остальных. Fix: 2-3 часа.

**2. Буквица — единая точка отказа (SPOF).**
- Глобальный `asyncio.Lock()` — одновременно обрабатывается 1 файл
- RESPONSE_TIMEOUT = 900с (15 минут) — при зависании вся очередь стоит
- 30ч/мес лимит — 30 часовых встреч, при 10 юзерах недостаточно
- Сессия "может протухнуть" — ручная перезаливка через fly ssh
- Рекомендация: OpenAI Whisper API как primary ($0.36/час)

**3. Zoom/API токены в открытом виде в БД.**
`zoom_access_token`, `zoom_refresh_token`, `user_*_api_key` — plain text в PostgreSQL. Zoom access_token передаётся в URL query parameter.

**4. Invite-only обходится.**
`/api/auth/register` не проверяет инвайт-код. OAuth-юзеры создаются без инвайта.

### P1 — Серьёзные

- Fly.io: фоновые задачи (Zoom poller) запускаются на обеих машинах → race condition
- Миграции без Alembic (18 ручных ALTER TABLE)
- JWT живут 30 дней без механизма отзыва
- SECRET_KEY с hardcoded fallback
- Rate limiting (slowapi) в deps, но не подключён

### Масштабируемость

| Метрика | 10 юзеров | 100 юзеров | 1000 юзеров |
|---|---|---|---|
| Буквица | Работает | **Упёрлись в лимит** | Невозможно |
| RAM 1GB | OK | На грани | **OOM** |
| Zoom polling | OK | **Rate limit** | Нужен webhook |
| LLM rate limits | OK | Groq/Gemini лимиты | Только платные API |

## CPO: Ключевые находки

### Два продукта, притворяющихся одним

- **Продукт A** (работает): персональный инструмент консультанта с вертикальными скиллами, AI-чатом по серии встреч, памятью
- **Продукт B** (амбиция): SaaS на 1000 пользователей при стеке, который не выдержит 50

**Приоритет: Продукт A.** Одна вовремя извлечённая задача из звонка с клиентом 20M+ стоит больше, чем 100 подписок.

### Онбординг — 7 шагов, 3-5% completion rate

Шаг 2 (Буквица) требует покупки подписки стороннего сервиса за 600₽/мес. Шаг 3 (агент) — `curl | bash` в терминале. Time-to-value: 20+ минут.

**Рекомендация:** 3 шага: приветствие → Zoom OAuth (1 клик) → Telegram (опционально). Демо-встреча при регистрации.

### Реальная ниша

Не "все кто зумит", а **русскоязычные солопренёры-консультанты**, ведущие клиентов через серию Zoom-звонков. TAM ~20-60M ₽/год.

### Настоящая дифференциация (не эксплуатируется)

1. Вертикальные скиллы (marketing_strategist с фреймворком "Знаю-Хочу-Верю-Плачу")
2. Память и база знаний per-user
3. Чат по папке встреч = анализ прогресса клиента за 3-6 месяцев
4. Русский язык нативно

### Рекомендации по фичам

**Убрать:** SupportTicket система, Email уведомления (SMTP), Google/Yandex OAuth, per-user Telegram Буквица
**Доделать:** Демо-встреча, Whisper для uploads, Zoom native transcript import
**Не трогать:** Fallback chain, AI-чат + streaming, скиллы, папки, per-user настройки AI

## Head of Growth: Ключевые находки

### Текущее состояние: Pre-launch, 0 demand

Invite-only при 0 юзерах — забор, не дефицит. K-factor = 0.6 (суб-виральный).

### Главный missing feature — share-ссылки

Публичная ссылка на саммари с "Powered by ZoomHub" — единственный path к виральности. Каждая встреча = 2-5 участников. При 40 юзерах = 64 новых регистрации/мес.

### Не хватает

- Лендинг (zoomhub.ru → сразу логин, конверсия ~2% vs 8-15%)
- Usage analytics (PostHog / аналог)
- Email-нотификации (Resend/Brevo, 300 писем/день бесплатно)

### План на 3 месяца

**Месяц 1:** Починить онбординг + Алмаз приглашает 20-30 людей → 10 активных юзеров
**Месяц 2:** Контент (vc.ru, Habr, TG) + share-ссылки → 40 активных
**Месяц 3:** Invite-only с waitlist (5 кодов на юзера, K=1.25) → 100 активных

### 152-ФЗ — стоп-фактор для B2B

Сервер в Амстердаме + записи переговоров. Решения: self-hosted Docker, VPS в РФ + proxy к LLM, GigaChat как единственный LLM для РФ-клиентов.

## Консенсус (все 4 эксперта согласны)

1. **Буквица не масштабируется.** Для SaaS — только Whisper API или аналог.
2. **Онбординг: с 7 шагов до 3.** Убрать Буквицу из обязательного пути. Демо-встреча при регистрации.
3. **Сначала internal tool + 5-10 beta-юзеров, не SaaS на 1000.**
4. **native_api.py — фикс user isolation перед любым запуском.** Критическая уязвимость.
5. **Алмаз — главный канал дистрибуции на первые 3 месяца.**
