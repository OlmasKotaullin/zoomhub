# AI-чат с настраиваемыми ролями

## Overview
AI-чат встроен в ZoomHub как полноценный модуль. Пользователь задаёт роль (system prompt) — характер, контекст бизнеса, правила поведения — и AI отвечает в рамках этой роли. Чат работает в двух режимах: по конкретной встрече (с контекстом транскрипта/саммари) и как общий ассистент по всему проекту (папке). Роль привязана к папке проекта и распространяется на все LLM-провайдеры.

## Requirements

### Functional Requirements
1. **Настройка роли (system prompt)** — textarea на странице настроек папки, где пользователь пишет промт с описанием роли AI
2. **Автогенерация промта** — кнопка "Generate": пользователь описывает бизнес в 2-3 предложения, AI генерирует полный структурированный system prompt
3. **Привязка роли к папке** — каждая папка проекта имеет свою роль. При открытии чата в контексте папки/встречи автоматически подгружается соответствующий промт
4. **Чат по встрече** — на странице встречи кнопка "Спросить AI" открывает чат с контекстом транскрипта + саммари этой встречи
5. **Общий чат по проекту** — отдельная страница `/chat` с контекстом всех встреч выбранной папки
6. **Выбор контекста** — галочками можно включить/исключить конкретные встречи из контекста чата
7. **Выбор LLM-провайдера** — в чате можно выбрать провайдера (Groq, Gemini, GigaChat, Claude) из тех, что настроены у пользователя
8. **Потоковый вывод** — ответы AI появляются посимвольно/потокенно (SSE streaming)
9. **История диалогов** — все диалоги сохраняются в БД, можно вернуться к любому
10. **Новый диалог** — кнопка "Новый чат" начинает чистый диалог с сохранением контекста роли

### Non-Functional Requirements
- Потоковая передача через SSE (Server-Sent Events) — без WebSocket
- Время до первого токена < 2 секунд
- История хранится в PostgreSQL
- Контекст ограничен лимитами провайдера (саммари вместо полных транскриптов при превышении)
- UI соответствует текущему дизайну ZoomHub (Tailwind + HTMX)

## Data Model

### Entities

**FolderRole** — роль AI для папки:
```
id: int (PK)
folder_id: int (FK → folders.id)
user_id: int (FK → users.id)
name: str — название роли ("Ассистент клиники", "Юрист")
system_prompt: text — полный system prompt
created_at: datetime
updated_at: datetime
```

**ChatSession** — сессия диалога:
```
id: int (PK)
user_id: int (FK → users.id)
folder_id: int (FK → folders.id, nullable)
meeting_id: int (FK → meetings.id, nullable)
title: str — автоматическое название из первого сообщения
llm_provider: str — какой провайдер использовался
created_at: datetime
updated_at: datetime
```

**ChatMessage** — сообщение в диалоге:
```
id: int (PK)
session_id: int (FK → chat_sessions.id)
role: str — "user" | "assistant"
content: text
tokens_used: int (nullable)
created_at: datetime
```

### Relationships
- User → FolderRole: 1:N (у каждой папки пользователя своя роль)
- User → ChatSession: 1:N
- Folder → ChatSession: 1:N (nullable — общий чат без папки)
- Meeting → ChatSession: 1:N (nullable — чат по встрече)
- ChatSession → ChatMessage: 1:N

### Storage
PostgreSQL (существующая БД ZoomHub). Автомиграция через `init_db()`.

## API Design

| Method | Path | Description | Request Body | Response |
|--------|------|-------------|-------------|----------|
| GET | `/chat` | Страница чата (HTML) | — | HTML |
| GET | `/chat/session/{id}` | Загрузить сессию | — | HTML partial |
| POST | `/api/chat/sessions` | Создать сессию | `{folder_id?, meeting_id?, provider}` | `{session_id}` |
| POST | `/api/chat/message` | Отправить сообщение (SSE stream) | `{session_id, message}` | SSE stream |
| GET | `/api/chat/sessions` | Список сессий пользователя | query: `folder_id?` | `[{id, title, created_at}]` |
| DELETE | `/api/chat/sessions/{id}` | Удалить сессию | — | 204 |
| GET | `/api/chat/role/{folder_id}` | Получить роль папки | — | `{name, system_prompt}` |
| POST | `/api/chat/role` | Сохранить роль папки | `{folder_id, name, system_prompt}` | `{id}` |
| POST | `/api/chat/role/generate` | Автогенерация промта | `{description, folder_id}` | SSE stream с промтом |
| GET | `/api/chat/context/{session_id}` | Текущий контекст (какие встречи включены) | — | `[{meeting_id, title, included}]` |
| POST | `/api/chat/context/{session_id}` | Обновить контекст (вкл/выкл встречи) | `{meeting_ids: []}` | 200 |

## Streaming (SSE)

### Backend
```python
@router.post("/api/chat/message")
async def chat_message(request: Request, ...):
    async def event_stream():
        async for chunk in provider.generate_stream(messages):
            yield f"data: {json.dumps({'content': chunk})}\n\n"
        yield f"data: {json.dumps({'done': True})}\n\n"

    return StreamingResponse(event_stream(), media_type="text/event-stream")
```

### Frontend
```javascript
const source = new EventSource('/api/chat/message', ...);
// Или через fetch + ReadableStream для POST:
const response = await fetch('/api/chat/message', {method: 'POST', body: ...});
const reader = response.body.getReader();
```

Используется `fetch` + `ReadableStream` (не `EventSource`), т.к. нужен POST с телом запроса.

## Context Builder

Сервис `chat_context.py` собирает контекст для LLM:

1. **System prompt** — роль из FolderRole (или дефолтная "Ты — полезный ассистент")
2. **Контекст встреч** — саммари выбранных встреч папки (по умолчанию все, пользователь может отключить)
3. **Контекст конкретной встречи** — если чат привязан к встрече, добавляется полный транскрипт
4. **История сообщений** — последние N сообщений сессии
5. **Обрезка по лимиту** — если контекст превышает лимит провайдера, используются только саммари (без транскриптов)

Порядок приоритета при обрезке:
1. System prompt (всегда)
2. Последнее сообщение пользователя (всегда)
3. Текущая встреча (транскрипт → саммари)
4. История диалога (новые → старые)
5. Саммари других встреч (новые → старые)

## Error Handling
- **Провайдер недоступен**: SSE отправляет `{error: "Сервис временно недоступен"}`, UI показывает уведомление с предложением сменить провайдера
- **Превышен лимит контекста**: автоматическая обрезка, пользователю показывается уведомление "Использованы только саммари (контекст превышен)"
- **Нет API-ключа для провайдера**: UI блокирует выбор этого провайдера с подсказкой "Настройте ключ в настройках"
- **Пустая роль**: используется дефолтный промт "Ты — полезный AI-ассистент. Отвечай на русском языке."
