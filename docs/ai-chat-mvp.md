# AI-чат ZoomHub MVP — минимальный продвинутый чат

## Overview
MVP AI-чата для ZoomHub по референсу Zoom AI Companion. Фокус: streaming + шаблоны-карточки + контекст встречи/папки. Без перегруза — только то, что реально нужно ЦА (диджитал-специалисты, консультанты, РОПы с 5-7 звонками в день). Решает главную боль: "после 7 звонков невозможно вспомнить детали, а руками заполнять таблицы — рутина".

## Что ZoomHub уже делает
- Авто-транскрипция (Буквица)
- AI-саммари (TL;DR + задачи + темы + инсайты)
- Чат по встрече (вкладка AI-чат, без streaming)
- Чат по папке (кнопка на странице папки)

## Что добавляем в MVP

### 1. Streaming (потоковый вывод)
- Ответ AI появляется потокенно, плавно, без дёрганья
- Debounce markdown render (50ms)
- Все 5 LLM-провайдеров поддерживают streaming
- Typing indicator пока ждём первый токен

### 2. Шаблоны-карточки (5 штук)
Заменяют текущие кнопки "Ключевые решения / План действий / Ответственные / Резюме" на более полезные:

| # | Шаблон | Промт для AI | Формат ответа |
|---|---|---|---|
| 1 | **Follow Up** | Определи задачи, ответственных и следующие шаги | ✅ Чеклист задач с ответственными |
| 2 | **Резюме встречи** | Дай структурированное резюме: темы, решения, задачи | 📋 Мета + 🧭 Темы + 🧩 Решения + ✅ Задачи |
| 3 | **Протокол** | Составь формальный протокол встречи | 📄 Повестка, участники, решения, сроки |
| 4 | **Трекер задач** | Извлеки все задачи, назначь ответственных и дедлайны | Таблица: задача / ответственный / срок / статус |
| 5 | **Отчёт за день** | Составь сводку по всем встречам за сегодня | Общий итог дня + по каждой встрече кратко |

Поведение при клике (как у Zoom):
- Промт подставляется в поле ввода
- Тег-бейдж появляется ("Follow Up ×")
- Контекст привязывается автоматически
- Кнопка отправки активируется
- Пользователь может отредактировать промт перед отправкой

### 3. Контекст (привязка данных)
- На странице встречи: контекст = текущая встреча (автоматически)
- На странице /chat: выбор через dropdown — папка целиком или конкретная встреча
- Тег-бейдж показывает привязанный контекст ("📅 Встреча × " или "📁 Папка ×")

### 4. Панель действий под ответом
- **📋 Копировать** — копирует markdown-ответ в буфер
- **🔄 Регенерировать** — повторный запрос к LLM

Без: оценок (👍👎), сохранения, thought process — это для следующей итерации.

### 5. Страница /chat (отдельная)
- Ссылка "AI-чат" в сайдбаре ZoomHub
- Минималистичный layout как у Zoom: поле ввода по центру + карточки-шаблоны под ним
- Dropdown для выбора контекста: папка или встреча
- После отправки — streaming ответ с панелью действий
- Можно продолжить диалог (follow-up вопросы)

### 6. Улучшенная вкладка на странице встречи
- Те же 5 карточек-шаблонов вместо текущих 4 кнопок
- Streaming вместо ожидания полного ответа
- Панель действий (копировать, регенерировать)
- Ссылка "Открыть в полном чате →"

## Чего НЕТ в MVP (следующие итерации)
- ❌ Кастомные шаблоны пользователя
- ❌ Настраиваемые роли / system prompts
- ❌ История сессий
- ❌ Оценки ответов (👍👎)
- ❌ Thought process
- ❌ Экспорт в Google Sheets / Excel
- ❌ Выбор LLM в чате (используется глобальная настройка)

## Requirements

### Functional Requirements
1. Streaming SSE для всех ответов AI-чата (и на /chat, и на вкладке встречи)
2. 5 встроенных шаблонов с предзаданными промтами и форматами ответа
3. Клик по карточке → промт в поле + тег-бейдж + автоконтекст
4. Выбор контекста на /chat: dropdown с папками и встречами
5. Панель действий: копировать (clipboard API) + регенерировать
6. Markdown рендеринг с emoji-заголовками
7. Дисклеймер "AI может ошибаться"
8. Typing indicator во время ожидания

### Non-Functional Requirements
- Время до первого токена < 2 сек
- Плавный streaming без дёрганья (debounce 50ms)
- Работает со всеми провайдерами (Groq, Gemini, Claude, GigaChat, Ollama)
- Русский язык по умолчанию
- Мобильный responsive (поле ввода + карточки адаптируются)

## Architecture

### Новые файлы
- `app/templates/chat.html` — страница /chat
- `app/services/chat_context.py` — сборка контекста для LLM

### Изменяемые файлы
- `app/services/providers/base.py` — `generate_stream()` в базовом классе
- `app/services/providers/{groq,gemini,claude,gigachat,ollama}_provider.py` — streaming
- `app/routers/chat.py` — новые эндпоинты (SSE, /chat страница)
- `app/templates/meeting.html` — новые карточки + streaming
- `app/templates/base.html` — ссылка AI-чат в сайдбаре

### API

| Method | Path | Description |
|--------|------|-------------|
| GET | `/chat` | Страница AI-чата (HTML) |
| POST | `/api/chat/stream` | Streaming сообщение `{message, meeting_id?, folder_id?, template?}` → SSE |

Формат SSE:
```
data: {"content": "токен"}\n\n
data: {"done": true}\n\n
```

### Системные промты шаблонов

Каждый шаблон = system prompt + user message. Формат ответа задаётся в system prompt:

```
Follow Up:
  "Ты анализируешь транскрипт встречи. Извлеки ВСЕ задачи, определи ответственных.
   Формат: чеклист ✅ с ответственным и сроком. Пиши на русском."

Резюме:
  "Структурированное резюме встречи. Секции: 📋 Мета (дата, участники),
   🧭 Темы обсуждения, 🧩 Решения, ✅ Задачи, 📌 Ключевые акценты.
   Используй emoji-заголовки, bold для ключевого. Русский язык."

Протокол:
  "Составь формальный протокол встречи. Секции: Повестка, Присутствовали,
   Обсуждение (по пунктам), Постановили, Сроки исполнения.
   Деловой стиль, русский язык."

Трекер задач:
  "Извлеки ВСЕ задачи из встречи. Формат таблицы:
   | Задача | Ответственный | Дедлайн | Статус |
   Если дедлайн не указан — напиши 'Не определён'. Русский язык."

Отчёт за день:
  "Сводка по всем встречам. Секции: 📊 Общий итог дня (2-3 предложения),
   затем по каждой встрече: название, ключевые решения, задачи.
   Кратко, по делу, русский язык."
```

## Verification
1. Открыть /chat → видны 5 карточек
2. Выбрать встречу в dropdown → тег-бейдж появился
3. Кликнуть "Follow Up" → промт подставился + тег шаблона
4. Нажать Send → streaming ответ, плавный вывод
5. Кнопка "Копировать" → текст в буфере
6. Кнопка "Ещё раз" → новый streaming ответ
7. На странице встречи → те же карточки + streaming
8. Проверить на всех провайдерах (Groq, Gemini)
