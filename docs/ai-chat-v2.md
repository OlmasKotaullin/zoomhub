# AI-чат ZoomHub v2 — по референсу Zoom AI Companion

## Overview
Продвинутый AI-чат для ZoomHub, вдохновлённый Zoom AI Companion. Работает в двух местах: отдельная страница `/chat` (основной интерфейс) и улучшенная вкладка AI-чат на странице встречи. Ключевые отличия от v1: готовые шаблоны-карточки, потоковый вывод (streaming), выбор контекста (папка/встреча), панель действий под ответом, история сессий. Без перегруза — минимализм Zoom, адаптированный под ЦА (диджитал-специалисты, консультанты, 5-7 звонков/день).

## Целевая аудитория
- Диджитал-специалисты: маркетологи, продуктологи, проджекты
- Консультанты: по 5-7 звонков в день, от 20 мин до 2 часов
- Потребности: быстро достать суть из встреч, не перечитывая стену текста

## Requirements

### Functional Requirements

**Страница /chat (основной интерфейс)**
1. Полноэкранный чат по референсу Zoom AI Companion — минималистичный, один экран
2. Поле ввода по центру с кнопками: `[+]` Add resources, `All sources` (выбор папки/встречи), `Send`
3. Шаблоны-карточки под полем ввода (табы: Suggested, Meeting, Custom)
4. При клике на карточку — промт подставляется в поле, тег-бейдж появляется, контекст привязывается автоматически
5. Выбор контекста: можно привязать папку целиком или конкретную встречу
6. Потоковый вывод ответа (SSE streaming) — плавный, без дёрганья
7. Панель действий под ответом: Копировать, Регенерировать, Оценить (👍/👎)
8. История сессий — сворачиваемый блок "History" в сайдбаре (как у Zoom)
9. Выбор LLM: по умолчанию авто, можно переключить в поле ввода

**Вкладка AI-чат на странице встречи**
10. Те же шаблоны-карточки (suggested prompts) вместо текущих кнопок
11. Потоковый вывод вместо ожидания полного ответа
12. Панель действий (копировать, регенерировать)
13. Ссылка "Открыть в полном чате" → `/chat?meeting_id=X`

**Шаблоны (карточки)**
14. Пользовательские кастомные шаблоны — пользователь может создать свой

### Non-Functional Requirements
- Streaming: время до первого токена < 2 сек, плавный вывод без дёрганья (debounce markdown render)
- Ответы на русском языке по умолчанию
- Markdown рендеринг: заголовки, списки, bold, чеклисты, разделители
- Emoji-заголовки в шаблонных ответах (📋, 🧭, ✅, ⚙️, 📌)
- Дисклеймер "AI может ошибаться" внизу
- Работает со всеми 5 LLM-провайдерами (Groq, Gemini, Claude, GigaChat, Ollama)

### Technology Stack
- Backend: FastAPI (существующий), SSE через StreamingResponse
- Frontend: HTMX + vanilla JS, Tailwind CSS (существующий стек)
- LLM: все 5 провайдеров через `generate_stream()` + fallback `generate()`
- БД: PostgreSQL (существующая)
- Streaming: SSE, `fetch()` + `ReadableStream` на клиенте

## Data Model

### Новые сущности

**ChatSession** — сессия диалога:
```
id, user_id (FK→users), folder_id (FK nullable), meeting_id (FK nullable),
title (String 500, auto из первого сообщения),
llm_provider (String 50 nullable),
created_at, updated_at
```

**ChatTemplate** — пользовательские шаблоны:
```
id, user_id (FK→users),
name (String 255), prompt (Text), category (String 50 default "custom"),
icon (String 10 default "💬"),
is_system (Boolean default False — для встроенных шаблонов),
created_at
```

### Изменения существующих моделей

**ChatMessage** — добавить:
- `session_id` (FK→chat_sessions, nullable — обратная совместимость)
- `template_name` (String 255, nullable — какой шаблон использовался)
- `tokens_used` (Integer, nullable)

## API Design

### Страница и сессии
| Method | Path | Description |
|--------|------|-------------|
| GET | `/chat` | Страница AI-чата (HTML) |
| POST | `/api/chat/sessions` | Создать сессию `{folder_id?, meeting_id?}` |
| GET | `/api/chat/sessions` | Список сессий (query: limit, offset) |
| DELETE | `/api/chat/sessions/{id}` | Удалить сессию + каскад |

### Streaming сообщение
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/chat/message` | `{session_id, message, provider?, template?}` → SSE stream |

Формат SSE:
```
data: {"content": "токен"}\n\n
data: {"content": "следующий токен"}\n\n
data: {"done": true, "message_id": 123}\n\n
```

### Шаблоны
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/chat/templates` | Все шаблоны (системные + пользовательские) |
| POST | `/api/chat/templates` | Создать кастомный `{name, prompt, icon?}` |
| DELETE | `/api/chat/templates/{id}` | Удалить кастомный шаблон |

## UI/UX: Ключевые паттерны из Zoom AI Companion

### 1. Поле ввода (центральный элемент)
- Большое textarea по центру страницы
- Сверху: теги контекста (📅 Встреча ×, 📁 Папка ×) — с крестиком для удаления
- Снизу: `[+]` добавить ресурс, `Sources` (выбор папки/встречи), `Mode` (LLM), `[→]` Send
- Placeholder: "Напишите сообщение..."

### 2. Шаблоны-карточки
- Табы: `Suggested` | `Meeting` | `Custom`
- Сетка 3×2 под полем ввода
- При клике: промт → textarea, тег шаблона → под textarea ("Follow Up ×")
- Карточка = название (bold) + описание (grey, 1-2 строки)

### 3. Ответ AI
- Markdown с emoji-заголовками (📋, 🧭, ✅, ⚙️)
- Разделители между секциями
- Bold для ключевой информации (даты, имена, решения)
- Панель действий: 📋 Copy | 🔄 Regen | 👍 👎

### 4. Streaming
- `fetch()` + `ReadableStream` (не EventSource, т.к. POST)
- Debounce markdown render (50ms) — плавность без дёрганья
- Typing indicator пока ждём первый токен

### 5. История
- Сворачиваемый блок "History ▸" в sidebar ZoomHub (под папками)
- Клик → список последних сессий (title + дата)
- Поиск по истории

### 6. Контекст
- Можно привязать папку целиком → AI анализирует все встречи
- Можно привязать конкретную встречу → AI работает с транскриптом
- Тег-бейдж показывает привязанный контекст (📅 Встреча × или 📁 Папка ×)

## Streaming Implementation

### Backend (SSE)
```python
@router.post("/api/chat/message")
async def chat_message(request, db):
    # 1. Сохранить user message
    # 2. Собрать контекст (встреча/папка)
    # 3. Создать провайдер
    # 4. generate_stream() → SSE
    async def event_stream():
        async for chunk in provider.generate_stream(messages, system=prompt):
            yield f"data: {json.dumps({'content': chunk})}\n\n"
        # Сохранить полный ответ в БД
        yield f"data: {json.dumps({'done': True, 'message_id': msg.id})}\n\n"
    return StreamingResponse(event_stream(), media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})
```

### Frontend (ReadableStream + debounce)
```javascript
const resp = await fetch('/api/chat/message', {method: 'POST', body: ...});
const reader = resp.body.getReader();
let fullText = '';
let renderTimer;

while (true) {
  const {done, value} = await reader.read();
  if (done) break;
  // parse SSE chunks → fullText += content
  // debounced render:
  clearTimeout(renderTimer);
  renderTimer = setTimeout(() => {
    el.innerHTML = marked.parse(fullText);
  }, 50);
}
```

## Системные промты для шаблонов

Каждый шаблон имеет system prompt на русском, который задаёт формат ответа:

```
Follow Up → структура: Задачи (чеклист), Ответственные (таблица), Следующие шаги
Резюме   → структура: Мета (дата, участники), Темы, Решения, Задачи, Ключевые акценты
Анализ   → структура: Прогресс, Невыполненные, Паттерны, Рекомендации
Отчёт    → структура: Сводка дня, Встречи (кратко), Задачи, Приоритеты
Протокол → структура: Повестка, Присутствовали, Обсуждение, Постановили, Сроки
```

Все промты требуют: markdown, emoji-заголовки, bold для ключевого, русский язык.
